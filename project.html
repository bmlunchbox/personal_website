StayTrack
Context
Patient care typically spans multiple departments, hospital floors, and sometimes days. Hospitals can be hectic and resultingly key pieces of information can be lost in transition. The standard methodology for tracking this information is using departmental, colour-coded whiteboards and spreadsheets. StayTrack was designed to consolidate them using a real-time, collaborative, and intuitive solution.
Development
At Oculys, I spent the majority of my time building microservices for the StayTrack platform. I worked with product managers to design storyboards and UIs that catered to the level of detail needed. This included preserving the original colours and formatting, while streamlining and standardizing the processes. I built modules to input and modify triage acuity scores, patient diagnoses, plans of care, various considerations, and therapy scheduling. This was built using C#, ASP.NET, JavaScript, and SignalR applying an MVC architectural pattern.
Trivia Bot
Context
Trivia is fun! HQ Trivia is a live mobile game where players are given 10 seconds per question and must answer all 12 correctly to win. I built a bot to determine the most probable answer.
Development
I used optical character recognition to extract the text and classified them using pattern matching. From there, I fed them through Google Analytics’ natural language processor to retrieve pronouns and assign salience scores. From there I produced multiple search queries ranging from just the question’s key words to combinations of the question and each query. Different scores are given to each results; some given for sheer quantity and others for answer matches. I’m in the process of implementing a gradient boost model so that the scores are dynamically assigned to queries based on question classification.
Disclaimer: I’ve never used this during a game… I’ve also never won.
Viewport
Context
At WalterFedy, there were two television monitors in the common area that played CP24 all day. Operations proposed a solution that would put them to better use while satisfying the business need for readily available and visually appealing KPIs. The spin-off benefits of this solution would be that it increased project transparency between departments and with upper management.
Development
The dashboard has a fixed framework and holds modular components on a rotational basis. Beyond the requested KPIs, the solution expanded to cover fun tidbits of company information, environmental impact assessments, Sunday comic strips, and local news. Local news drew from sanitized Twitter RSS feeds, while the rest of the information came from a SQL database. The dashboard was primarily built using JavaScript using Vue as its framework and D3.js for the data visuals.
Asset Tracker
Context
Keeping track of company technology, furniture, and machinery was done separately and inconsistently by different departments. A solution would be needed to centralize existing data sets, provide a means to add new ones, and generate department-specific reports for accounting and IT.
Development
After numerous iterations of research and proposal, building a solution around QR codes seemed like the cheapest option while still satisfying base requirements. The codes were generated using Google’s API and attached to each asset. I built a QR decoder using OpenCV in Python, which was then wrapped with C# in order for it to integrate with the rest of the solution. To democratize the data stored in SQL, I created portals with a two-way sync between Excel and SQL that would allow users to access and edit data from either end.
Nutrition Information Parser
Context
I found that the most tedious part of tracking my caloric intake was inputting the values from nutrition labels. I wanted to make an app that would recognize nutrition labels and extract relevant information to be processed later.
Development
The simple program used computer vision techniques like line detection, box creation, and optical character recognition to detect and upload the information of interest. Built in Python, using OpenCV and PyTesseract.
